# I am attaching information
# about Lake Superior runoff data. Note that this is from my 2012 update, so it
# doesn't reflect the current state of data availability. Primarily, the values
# from 2011-2012 should be ignored, since I had no Canadian runoff data. But it at
# least shows you what I can provide (once I get this data set updated as far as
# possible), and gives a historical sense.
# 
# Note that my runoff data does NOT use all available gauges. Our procedure is to
# first identify gauges that we might want to include, based on their location
# (not too close to lake, so we avoid backwater effects) and period of record.
# That gives us several dozen gauges per lake. Then, the ARM method only uses the
# most-downstream gauge from any branch of a river. Looking at the data (% gaged
# and number of gages) it might appear incongruous at times, since the number of
# gages may rise when the % falls. But that would be because a downstream gage did
# not report, so upstream gages on two separate branches were then used.
# 
# Note that the file for number of gauges in a month accounts for gaguges that
# report for only some days. This the fractional values. I think the data files
# should be relatively self-explanatory, but let me know if you have any questions
# about the files or methodology.
# 
# Tim
